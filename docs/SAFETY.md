# Safety

DesktopVisual 1.0.0 is for authorized local Windows desktop workflows.

Core boundaries:

- Use visible-first operation.
- Require explicit target windows.
- Use target-window-locked real input.
- Stop on failed locators, ambiguous windows, missing windows, and safety denials.
- Do not bypass protected desktops, elevated windows, credentials, CAPTCHA, anti-cheat, anti-automation, proctoring, payment, banking, or security controls.
- Do not use DesktopVisual for unauthorized third-party workflows.

Accepted visible input evidence does not rely on clipboard or backend writes.
